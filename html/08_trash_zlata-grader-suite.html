Docs layout

This section is written as a clean product guide for the multi-story grading and ranking workflow. It can be pasted directly into your future app as a help panel, onboarding screen, or standalone documentation page.

What the suite does

The grader workflow turns each story into a structured record with metadata, rubric scores, notes, tags, and a weighted Zlata-match score. The ranking board then stores multiple graded records so they can be sorted and compared as a growing corpus.

Quick start

  • Upload a local TXT or RTF story file.
  • Review the imported text for formatting or OCR issues.
  • Enter metadata such as title, source, and URL.
  • Grade the story against the rubric.
  • Add notes, tags, and warnings.
  • Save to the ranking board and export JSON.

Recommended workflow

  • Grade one story at a time.
  • Always verify imported text before scoring.
  • Use the rubric consistently across the corpus.
  • Treat machine extraction as a draft, not final authority.
  • Write short notes explaining why the score makes sense.
  • Export often so your corpus remains portable.

Ranking logic

High-scoring records with strong notes are likely Tier 1 matches. Mid-range records with caveats often belong in Tier 2 or adjacent categories. Low-scoring records or stories with disqualifying traits should be treated as false positives, different sub-genres, or useful-but-non-core comparators.

Perplexity Computer prompt

This prompt is written for Perplexity Computer so it can read a single story, extract consistent rubric values, and produce a clean JSON object that fits the grader and ranking workflow.

You are helping build a structured Zlata-style fiction corpus. Your task is to read one uploaded story and extract a grading record that matches a browser-based grading tool and multi-story ranking board. Follow these rules exactly: 1. Read the full uploaded story text carefully. 2. Do not summarize loosely. Extract structured values. 3. Judge the story against a Zlata-style corpus focused on these traits: - Everyday or normal man discovers or becomes fascinated by an extremely flexible woman. - Flexibility is the erotic center, not just a minor detail. - Sensual or erotic tension is built through demonstrations, passive stretching, training, voyeuristic observation, and chemistry. - Extreme positions matter: oversplits, backbends, triple folds, legs-behind-head, shoulder dislocations, packing, throughbends, or related flexibility vocabulary. - Consensual romantic or erotic framing is strongly preferred. 4. Flag major deviations clearly, such as magic, body-horror, coercion, detached fetish mechanics, or stories where flexibility is incidental rather than central. 5. Be consistent and conservative. If something is uncertain, say so in notes rather than inflating the score. Return your output in two parts. PART 1: SHORT ANALYSIS Provide a short structured analysis with these headings: - Core premise - POV and dynamic - Flexibility emphasis - Erotic tone - Best-matching Zlata elements - Major mismatches or warnings - Final judgment PART 2: JSON OBJECT Return one clean JSON object using this schema: { "title": "", "author": "", "source": "", "url": "", "estimated_length": "short|medium|long|unknown", "pov": "male_first|male_third|female_first|female_third|mixed|unknown", "setup_type": "discovery|dating|professional|training|performance|domestic|other", "male_discovery_dynamic_score": 0, "female_flexibility_centrality_score": 0, "technical_move_vocabulary_score": 0, "passive_training_or_assisted_stretching_score": 0, "voyeuristic_fascination_score": 0, "romantic_or_erotic_chemistry_score": 0, "consensual_tone_score": 0, "writing_quality_score": 0, "zlata_style_match_score": 0, "tier_guess": "tier1|tier2|adjacent|false_positive", "key_moves": [], "key_tags": [], "warnings": [], "notes": "", "one_sentence_verdict": "" } Scoring guidance: - Use a 0 to 10 scale for each score. - Reserve 9 to 10 for very strong examples. - Use low scores when the trait is absent, weak, or structurally unimportant. - zlata_style_match_score should be your overall weighted judgment, not a simple average. Important constraints: - Do not invent bibliographic details. - If title, author, source, or URL are unavailable, use an empty string. - warnings should contain short machine-readable phrases like "magic", "coercive_dynamic", "body_horror", "non_core_pov", "flexibility_incidental", or "uncertain_metadata". - key_moves should list specific flexibility elements actually present in the text. - key_tags should be concise searchable tags. - notes should explain the score and mention the single biggest reason the story does or does not match the corpus.
